# 《基于RAG的中国传统诗词场景可视化系统》详细架构设计与需求规格说明书

## 1. 引言

### 1.1 编写目的

本说明书旨在全面、系统地描述“基于RAG的中国传统诗词场景可视化系统”  的业务需求、系统架构、数据流转及接口规范。本文档将作为前后端分离开发、微服务对接及最终系统集成测试的最高技术准则。

### 1.2 业务痛点分析

当前主流文本到图像（T2I）生成模型（如Stable Diffusion）由于缺乏对中国古典文言文及特有意象的深度语义对齐，在处理凝练的古典诗句时，常出现关键意象缺失、文化符号误用、画面意境与诗意不符等问题 。传统直接输入诗句（Base Line）的生成方式无法满足数字文化传播对高度还原传统美学特征的要求。

### 1.3 核心系统架构思想

系统引入检索增强生成（RAG）范式 。其核心技术路径为：弃用大语言模型（LLM）的高成本实时推理重写，转而采用“前置数据清洗+精准语义检索+模板化微调（Prompt Engineering）”的极简高效策略。系统通过检索古诗对应的现代白话文译文与意象解析 ，结合预设的国风美学控制词，驱动挂载特定国风LoRA的图像生成模型 ，实现高质量、高确定性的诗意可视化。

## 2. 总体逻辑架构设计

系统采用**面向服务架构（SOA）**理念，基于“前后端分离 + 算法微服务化”的解耦模式，总体分为三层：

### 2.1 表现层 (Presentation Layer)

* **技术栈：** Vue 3 (Composition API) + Vite + Element Plus。
* **核心功能：** 负责客户端的视图渲染与状态管理。包含鉴权模块、流式任务状态轮询机制、可视化RAG链路展示（输入->检索内容->增强提示词->最终生成图像的联动展示）及作品画廊。

### 2.2 业务逻辑与调度层 (Business Logic & Dispatch Layer)

* **技术栈：** Spring Boot 2.7+ / 3.x + MyBatis-Plus。
* **核心功能：** 作为系统的网关与中枢控制器。处理高并发下的HTTP请求，管理MySQL中的系统元数据与用户状态；**承担异步任务调度中心的角色**，避免阻塞式IO等待。

### 2.3 AI算法微服务层 (AI Algorithm Microservice Layer)

* **技术栈：** Python 3.10+ + FastAPI + LangChain + Diffusers。
* **核心功能：** 封装所有高耗时的计算密集型任务。内部细分为：
* 
**向量引擎模块：** 运行ChromaDB，管理和检索文本向量 。


* 
**语义检索模块：** 调用 `bge-large-zh-v1.5`  提取特征向量，执行余弦相似度（Cosine Similarity）匹配。


* 
**推理引擎模块：** 挂载Stable Diffusion基底模型与国风LoRA权重 ，执行T2I生成。

## 3. 核心业务流程与状态机设计

为解决大模型推理耗时导致的网络超时异常，系统级数据流转必须严格遵循**异步轮询任务状态机**模型。

### 3.1 核心生图时序逻辑

1. **任务投递：** 客户端发起包含源诗句的 `POST` 请求至 Spring Boot。
2. **持久化与派发：** Spring Boot 在 MySQL 中生成 `task_id`，状态初始化为 `PENDING(0)`。随后将任务载荷通过内部 HTTP 请求派发至 Python FastAPI 节点。
3. **微服务接管：** FastAPI 接收请求，将其放入 `BackgroundTasks` 守护线程池中，并立即向 Spring Boot 响应 `HTTP 200 (Accepted)`。Spring Boot 将 `task_id` 返回给客户端。
4. **客户端轮询：** 客户端基于 `task_id` 启动定时器（如周期为3秒），向 Spring Boot 轮询任务最新状态。
5. **算法链执行（核心后台流）：**
* Python 服务调用 Embedding 模型向量化源诗句 。


* 在 ChromaDB 中检索出 Top-1 最匹配的现代白话文译文。
* 字符串拼接：`Final Prompt = {白话文译文} + {风格化固定Prompt(如: 中国传统水墨, 青绿山水, 8k)}`。
* 扩散模型（SD）基于 `Final Prompt` 执行去噪生成 。


* 图像输出至本地或 OSS（对象存储），获得 `Image_URI`。


6. **结果回调：** Python 服务向 Spring Boot 发送 Webhook 回调，携带 `Image_URI`、`Final Prompt` 及 `译文文本`。
7. **状态流转：** Spring Boot 更新 MySQL 对应记录，状态流转为 `COMPLETED(1)`。客户端下一次轮询获取到完整数据，终止定时器并渲染视图。

## 4. 详细物理数据模型设计 (Database Schema)

### 4.1 关系型数据库 (MySQL) - 业务层支持

**表名：`sys_generation_task` (生图任务流转表)**

* **设计原则：** 记录生图全生命周期，支持业务侧的历史追溯。

| 字段名 (Column) | 数据类型 (Type) | 约束 (Constraints) | 默认值 | 业务说明 (Description) |
| --- | --- | --- | --- | --- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 物理主键 |
| `task_id` | VARCHAR(64) | UNIQUE, INDEX | - | 分布式任务追踪ID (UUID) |
| `user_id` | BIGINT | INDEX | - | 关联提交任务的用户ID |
| `original_poem` | VARCHAR(255) | NOT NULL | - | 用户输入的原始古诗句 |
| `retrieved_text` | TEXT | NULL | - | RAG检索出的现代文译文/知识 |
| `enhanced_prompt` | TEXT | NULL | - | 最终注入SD模型的正向提示词 |
| `result_image_url` | VARCHAR(512) | NULL | - | 生成完成后的图像访问相对/绝对路径 |
| `task_status` | TINYINT | NOT NULL | 0 | 任务枚举状态：0-排队/生成中, 1-成功, 2-失败 |
| `error_message` | VARCHAR(512) | NULL | - | 失败时的异常堆栈或原因简述 |
| `gmt_create` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 记录创建时间 |
| `gmt_modified` | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 记录最后更新时间 |

### 4.2 向量数据库 (ChromaDB) - 算法层支持

**集合名称 (Collection)：`poetry_modern_translation_knowledge**`

* 
**Embedding Model：** `BAAI/bge-large-zh-v1.5` (输出维度：1024) 


* **Distance Function：** 余弦距离 (Cosine)
* **数据结构定义：**
* `id`: 唯一哈希值（如 `MD5(原始诗句)`）
* `document`: (核心检索区) 古诗句的现代白话文详细译文及画面白描。
* `metadata`: 包含 `{"dynasty": "朝代", "author": "作者", "original_text": "原文句"}`，用于业务层的二次校验或硬过滤。

## 5. 核心跨层 API 接口契约规范 (RESTful)

### 5.1 Web前端 -> Java后端：创建任务接口

* **Endpoint:** `POST /api/v1/poetry/visualize`
* **Request Body (JSON):**
```json
{
  "poemText": "大漠孤烟直，长河落日圆"
}

```


* **Response (JSON):**
```json
{
  "code": 200,
  "message": "Task submitted successfully",
  "data": {
    "taskId": "7f8b9-xxxx-xxxx"
  }
}

```



### 5.2 Java后端 -> Python微服务：指令下发接口

* **Endpoint:** `POST /ai/api/v1/generate/async`
* **Request Body (JSON):**
```json
{
  "taskId": "7f8b9-xxxx-xxxx",
  "sourceText": "大漠孤烟直，长河落日圆",
  "callbackUrl": "http://java-backend-ip:8080/api/v1/poetry/callback"
}

```


* **Response:** `HTTP 202 Accepted` (表示任务已进入Python后台队列)。

### 5.3 Python微服务 -> Java后端：结果回调接口

* **Endpoint:** `POST /api/v1/poetry/callback`
* **Request Body (JSON):**
```json
{
  "taskId": "7f8b9-xxxx-xxxx",
  "status": 1,
  "payload": {
    "retrievedText": "浩瀚沙漠中孤直的烽烟，黄河边一轮浑圆的落日。",
    "enhancedPrompt": "浩瀚沙漠中孤直的烽烟，黄河边一轮浑圆的落日。中国传统水墨画，青绿山水，极高分辨率，8k，杰作。",
    "imageUrl": "/statics/outputs/7f8b9-xxxx.png"
  }
}

```

## 6. 系统非功能性需求与性能指标 (NFR)

1. **容错与熔断机制：** 当Python AI微服务显存溢出（OOM）或生成时间超过系统预设阈值（如120秒）时，FastAPI需主动捕获异常并向Java后端发起失败回调，避免前端无限轮询（死锁）。
2. **资源约束限制：** Diffusers 推理管线初始化时，应开启 `enable_xformers_memory_efficient_attention()` 以及 `enable_model_cpu_offload()`，以适应单卡GPU（如RTX 3060/4060等消费级显卡）的显存限制。
3. **静态资源服务器：** AI生成的图片采用Nginx进行反向代理映射，Java业务端仅存储文件相对路径，实现动静分离。
